# Template — Single Tweet Launch Announcement

**Use for:** YC-style day-of launches. AI-era demo-first launches. Founders with existing audience. Products where the video does most of the persuasion.

**Length:** 1 tweet (≤280 chars on X) + link in REPLY tweet.

---

## Fill-in skeleton

```
Today, we're launching [PRODUCT] — [ONE-LINE VALUE PROP, 5-8 WORDS].

[PROBLEM, 1 LINE, WITH ADJECTIVE THAT TWISTS THE KNIFE].

[SOLUTION + QUALITY GUARD, 1 LINE].

[Video embedded natively]
```

Reply tweet (1st reply):
```
Try it: [URL]
```

---

## Worked example (Snowglobe-style)

```
Introducing ❄️ Snowglobe, the simulation engine for AI chatbots.

Finding blind spots by the time your users do is brutally late.

Snowglobe simulates thousands of realistic user conversations so you catch failures before they ship.

[Video]
```

Reply:
```
Try it: snowglobe.so
```

---

## Variants by archetype

### AI-era demo-first (Bolt/v0 style)
Swap the 3-line structure for a SINGLE QUESTION:
```
what do you want to build?

[Video]
```

### Fundraise announcement (Osika $200M style, short version)
```
Today [Company] raised $[X] at a $[Y] valuation led by [Lead VC].

[Video or team photo]
```
Full origin story goes in a long-tweet version (see `fundraise-long-post.md`).

### Benchmark launch (Devin style)
```
Today we're excited to introduce [Product], the first [Category].

[Product] correctly [does X] [N]% of the time — far exceeding the previous state-of-the-art of [Y]%.

[Video]
```

---

## Pre-ship checklist

- [ ] First 10 words contain product name + category
- [ ] Video embedded natively (not YouTube link)
- [ ] Link is in REPLY, not main tweet
- [ ] No em dashes
- [ ] No "excited to share" / "stay tuned"
- [ ] Published Tue/Wed/Thu 9am PT
- [ ] Account warmed up 3-5 days prior
- [ ] Team pinged to repost
