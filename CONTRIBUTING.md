# Contributing

Thanks for considering a contribution. This repo is a living playbook — every new viral launch teaches us more about the formula.

## What to contribute

### 1. Add a new launch to the corpus

If you find a great viral launch (your own or someone else's), add it.

**Template:** copy `corpus/01-crunched.md`. Fill in:
- YAML frontmatter (company, founder, platform, date, URL, engagement, archetype)
- Verbatim post text (in a blockquote)
- Pattern analysis (hook, structural beats, signature tics)
- Lessons for the skill

Open a PR. We'll merge if the post is genuinely instructive.

### 3. Improve the references

If you have evidence (data, A/B tests, your own launch experience) that a rule in `references/` is wrong or incomplete, open an issue or PR. Cite the source.

### 4. Translate

The repo is English-only right now. Translations welcome.

## What NOT to contribute

- ❌ Speculation. Every claim should be sourced to a real launch.
- ❌ Self-promo posts. Adding your own launch is fine ONLY if it's genuinely instructive (and you're honest about engagement, including failures).
- ❌ AI-generated launch posts presented as real. The corpus is verbatim only.
- ❌ Tactics for inauthentic engagement (bot networks, fake comments, etc.).

## Style guide

- No em dashes (the rule applies to the docs themselves)
- Use the 5-beat formula language consistently
- Be specific over abstract — use real names, real numbers, real dates
- Mark uncertainty explicitly ("not retrievable verbatim", "engagement unknown")
