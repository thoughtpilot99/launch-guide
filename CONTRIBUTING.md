# Contributing

This repo gets better the more launches it documents. If you have one to add, send it.

## What to contribute

### Add a new launch to the corpus

Find a viral launch (yours or someone else's). Add it.

Copy `corpus/01-crunched.md`. Fill in:
- YAML frontmatter (company, founder, platform, date, URL, engagement, archetype)
- Verbatim post text in a blockquote
- Pattern analysis (hook, structural beats, signature tics)
- Lessons for the skill

Open a PR. If the post is genuinely instructive, we merge.

### Improve the references

If you have evidence (data, A/B tests, your own launch experience) that a rule is wrong or incomplete, open an issue or PR. Cite the source.

### Translate

English-only right now. Translations welcome.

## What NOT to contribute

- Speculation. Every claim should be sourced to a real launch.
- Self-promo posts. Adding your own launch is fine ONLY if it's instructive (and you're honest about engagement, including failures).
- AI-generated launch posts presented as real. The corpus is verbatim only.
- Tactics for inauthentic engagement (bot networks, fake comments).

## Style guide

- No em dashes. The rule applies to the docs themselves.
- Use the 5-beat formula language consistently.
- Specific over abstract. Real names. Real numbers. Real dates.
- Mark uncertainty explicitly ("not retrievable verbatim", "engagement unknown").
